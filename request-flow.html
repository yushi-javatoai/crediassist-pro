请求处理详细流程 

sequenceDiagram
    participant User as 用户
    participant API as API层
    participant Supervisor as Supervisor
    participant Intent as IntentRouter
    participant Agent as 子Agent
    participant Memory as 记忆系统
    participant Compliance as 合规审查
    
    User->>API: 发送消息
    API->>Memory: 存储用户消息
    API->>Supervisor: 转发请求
    Supervisor->>Intent: 意图识别
    Intent-->>Supervisor: 返回意图分类
    Supervisor->>Agent: 路由到对应Agent
    
    alt 知识检索
        Agent->>Memory: 查询长期记忆
        Memory-->>Agent: 返回相关文档
        Agent->>Agent: 生成回答
    else 工单处理
        Agent->>Agent: 处理工单
    else 其他任务
        Agent->>Agent: 处理任务
    end
    
    Agent-->>Supervisor: 返回处理结果
    Supervisor->>Compliance: 合规审查
    Compliance-->>Supervisor: 审查结果
    Supervisor->>Memory: 存储对话历史
    Supervisor-->>API: 汇总最终结果
    API-->>User: 返回响应
1

接收请求

FastAPI接口接收用户消息，存储到短期记忆，转发给Supervisor

2

意图识别

IntentRouter分析用户意图，进行一级和二级分类，提取关键实体

3

智能路由

Supervisor根据意图类型，将请求路由到对应的子Agent

4

Agent处理

子Agent处理具体任务：知识检索、工单处理、合规审查等

5

合规审查

两阶段审查：规则引擎快速检查 + LLM深度审查

6

结果返回

Supervisor汇总结果，存储到记忆系统，返回给用户

技术亮点

多Agent协作

Supervisor编排多个专业Agent，实现任务的智能分配和协作

三级记忆系统

工作记忆 + 短期记忆(Redis) + 长期记忆(FAISS)，提供完整的记忆能力

端到端RAG流程

Query改写 → 向量检索 → 重排序 → 上下文注入 → 生成回答

金融级合规

两阶段审查(规则引擎 + LLM深度审查)，确保回答符合监管要求

核心组件

组件名称 主要功能 技术实现
Supervisor 中央协调者，负责意图路由和结果汇总 LangGraph StateGraph
IntentRouter 意图识别与分类，实体提取 LLM + 规则引擎
KnowledgeRAG 知识库检索与回答生成 FAISS + LangChain
TicketHandler 工单创建与查询，状态管理 内存存储 + 优先级算法
ComplianceChecker 合规审查，敏感词检测，PII脱敏 规则引擎 + LLM深度审查
Memory System 三级记忆管理 进程内 + Redis + FAISS
