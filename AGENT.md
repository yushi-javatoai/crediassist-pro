# Agent 管理规范

## 1. 项目概述

本项目是一个基于Agent的信用辅助系统（CrediAssist Pro），使用Python实现，包含多个功能模块和Agent组件。项目结构如下：

```
crediassist-pro/
├── python-impl/
│   ├── frontend/         # 前端应用
│   ├── agents/           # Agent组件
│   ├── api/              # API接口
│   ├── mcp/              # 工具和服务
│   ├── memory/           # 内存管理
│   ├── tracing/          # 追踪功能
│   ├── requirements.txt  # 依赖管理
│   └── Dockerfile        # 容器配置
├── docker-compose.yml    # 容器编排
├── README.md             # 项目说明
└── .gitignore            # Git忽略文件
```

## 2. 代码规范

### 2.1 Python 代码规范

- **遵循 PEP 8 规范**：使用 4 个空格进行缩进，每行不超过 79 个字符
- **命名规范**：
  - 类名：使用驼峰命名法（CamelCase）
  - 函数和变量名：使用小写字母和下划线（snake_case）
  - 常量：使用全大写字母和下划线
- **文档规范**：
  - 每个模块、类和函数都应有文档字符串（docstring）
  - 文档字符串使用 Google 风格
- **导入规范**：
  - 按标准库、第三方库、本地模块的顺序导入
  - 使用绝对导入而非相对导入

### 2.2 代码风格检查

- 使用 `flake8` 进行代码风格检查
- 使用 `black` 进行代码格式化
- 使用 `isort` 进行导入排序

## 4. 开发流程

### 4.1 分支管理

- `main`：主分支，只包含稳定版本
- `develop`：开发分支，集成所有功能开发
- `feature/*`：功能分支，用于开发新功能
- `bugfix/*`： bug 修复分支，用于修复 bug

### 4.2 提交规范

提交信息应遵循以下格式：

```
<类型>: <描述>

<详细说明>
```

类型包括：
- `feat`：新功能
- `fix`： bug 修复
- `docs`：文档更新
- `style`：代码风格调整
- `refactor`：代码重构
- `test`：测试相关
- `chore`：其他更改

### 4.3 代码审查

- 所有代码变更都应通过代码审查
- 代码审查应关注代码质量、安全性和性能

## 5. 测试规范

### 5.1 测试类型

- **单元测试**：测试单个函数或方法
- **集成测试**：测试多个组件的交互
- **端到端测试**：测试整个系统的功能

### 5.2 测试框架

- 使用 `pytest` 进行测试
- 测试文件应放在 `tests` 目录中
- 测试文件命名应为 `test_*.py`

### 5.3 测试覆盖率

- 代码覆盖率应达到 80% 以上
- 使用 `coverage.py` 进行覆盖率分析
